## $ whoami

This is my personal portfolio site, built to reflect what I actually do — DevOps engineering, cloud infrastructure, and Linux automation — rather than what most portfolio templates assume (a generic dark-mode React app with hero text and a contact form).

The entire site is a **single `index.html` file**. No framework. No build step. No node_modules. Just deploy and go.

## $ ls -la ./features

```
drwxr-xr-x  boot sequence       Linux kernel-style startup animation on load
drwxr-xr-x  about.yaml         Profile rendered as a parsed YAML config file
drwxr-xr-x  git log            Work experience as a git commit graph
drwxr-xr-x  htop               Skills displayed as animated CPU/memory bars
drwxr-xr-x  ls -la             Projects as an expandable file listing
drwxr-xr-x  /etc/certs         Certifications as system config entries
drwxr-xr-x  ssh contact        Contact section styled as an SSH connection
-rwxr-xr-x  uptime ticker      Live counter running from first day at Accenture
```

## $ cat stack.txt

- Pure HTML5, CSS3, Vanilla JS — zero dependencies
- [JetBrains Mono](https://fonts.google.com/specimen/JetBrains+Mono) for that real terminal feel
- GitHub Public API for live activity data
- Deployed on [Vercel](https://vercel.com) with automatic redeploy on every push

## $ git clone && deploy

```bash
# clone
git clone https://github.com/CaptainPatel/portfolio.git
cd portfolio

# that's it — open index.html in your browser
# or deploy to vercel in one click
```
